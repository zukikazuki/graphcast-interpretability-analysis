# 研究内容まとめ：GraphCastの推論機序を「操作と観測」で解明する

この研究は、GraphCastをブラックボックスとして扱いつつも、**モデル内部の情報伝播（message passing）**と**出力の振る舞い**の対応関係を、
「どの内部要素を操作し（介入し）」「何を観測し」「どう解釈するか」という形で整理し、
GraphCastの推論機序（どのように予報場が作られるか）を説明可能にすることを狙います。

---

## 研究の問い（ブラックボックス解剖の方針）

- **問い**: GraphCastは、全球の格子データから、どのような計算過程で次の大気状態を生成しているのか？
  - どのスケールの情報（局所〜全球）が、どの変数・どの高度（気圧面）に効いているのか？
  - 内部の情報伝播回数を減らす/増やすと、どの物理量の予測が先に崩れ、どれが頑健なのか？

---

## GraphCastの推論機序（コードに基づく“機械としての説明”）

GraphCastの1ステップ推論は、概ね次の3段に分かれます。

### 1) Grid → Mesh（エンコード：格子からメッシュへ情報を移す）

格子点（lat/lon）をノードとし、球面上の三角メッシュ（icosahedral mesh）のノードへ、近傍半径に基づくエッジで情報を送ります。
この段は **1回のmessage passing**で、格子の観測（入力）をメッシュ上の潜在表現（latent）に写します。

### 2) Mesh上での反復 message passing（処理器：推論の“計算の核”）

メッシュノードだけで構成されるグラフ上で、**`gnn_msg_steps` 回だけ message passing を反復**します。
ここが「情報が球面上を何ステップ伝播するか」を決める、推論機序の中心です。

- `gnn_msg_steps` を小さくする：局所的な情報交換が少なく、**長距離の依存**（大規模循環や広域の結びつき）が表現しにくくなる可能性
- `gnn_msg_steps` を大きくする：情報統合の反復が増え、広域の整合が取りやすくなる一方、計算コストは増える

### 3) Mesh → Grid（デコード：メッシュから格子へ戻して予測場を出す）

更新されたメッシュ潜在表現を、格子点に戻して各変数の予測を出力します。
この段も **1回のmessage passing**で、メッシュ表現を格子上の予測（次時刻の大気場）へ投影します。

---

## “時間方向”の推論機序：自回帰ロールアウト

GraphCastは「1ステップ先」を予測するモデルを、**自回帰（autoregressive）**に繰り返して複数ステップ先を作ります。
このとき、あるステップで得た予測が次ステップの入力に**フィードバック**され、
入力が複数時刻（例：過去12時間の履歴）で与えられる場合は、**ローリングウィンドウ**で古いフレームを落として新しい予測を足します。

その結果、`gnn_msg_steps` の効果は「1ステップ先」だけでなく、ロールアウトで**誤差が累積する仕方**にも表れます。

## 解明アプローチ（この研究での“介入”と“観測”）

### 介入：`gnn_msg_steps` を振る（計算の深さ＝情報伝播回数の操作）

`gnn_msg_steps` はメッシュ上の反復 message passing 回数であり、
推論機序の中で「どれだけ反復して空間的に情報を混ぜるか」を直接操作できます。
ブラックボックス解析としては強い介入で、次のような解釈が可能になります。

- **どの変数が“反復を必要とする”か**（=広域統合・非局所相互作用に依存しやすいか）
- **どの高度で効くか**（気圧面ごとの感度）
- **どの程度の反復で飽和するか**（追加の反復が誤差低減に寄与しなくなる点）

### 観測：誤差だけでなく「予測分布の形」を見る

単にRMSEを見るだけでなく、

- 予測値の平均（pred_mean）
- 分散/標準偏差（pred_std）
- 変動係数（pred_cv）

などを併用し、`gnn_msg_steps` によって **出力のスケール感・ばらつきがどう変わるか**を観測します。
これは「当たる/外れる」以外に、モデルが作る場の統計的性質（過平滑化、過大振幅、バイアスの兆候など）を捉えるためです。

---

## この研究で得たい“機序の説明”の形（アウトカム）

最終的に、次のような説明を目指します。

- GraphCastは **「格子→メッシュ→（反復伝播）→格子」** という3段で推論する
- 中核である反復伝播回数（`gnn_msg_steps`）を変えると、
  - 影響が大きい変数
  - 影響が大きい高度帯（どの気圧面が壊れやすいか）
  - 影響が出やすい統計的性質（過小/過大分散、バイアスなど）
  が系統的に現れる
- したがってGraphCastのブラックボックス性は、
  **「反復が担う役割（局所→広域の統合、変数間・高度間のカップリング）」**という観点で分解できる
